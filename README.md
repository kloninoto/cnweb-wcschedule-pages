# cnweb-wcschedule-pages

这是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定网站或域名，仅作为静态页面的集中存放与维护用途。

## 项目简介

本项目的主要目的是收集、整理并托管一批独立的 HTML 页面。这些页面各自独立，互不依赖，适用于需要以纯静态方式呈现内容或功能的场景。仓库本身不包含后端服务、数据库或动态逻辑，所有页面均可直接通过浏览器访问。

## 目录说明

```
.
├── index.html          # 仓库首页/页面导航（可选）
├── page1.html          # 示例页面1
├── page2.html          # 示例页面2
└── assets/             # 公共资源目录
    ├── css/            # 样式文件
    └── js/             # JavaScript 文件
```

目录结构可能随页面数量增加而调整。每个 HTML 页面通常位于根目录下，公共资源统一存放于 `assets` 文件夹内。

## 页面归档说明

本仓库中的 HTML 页面均以独立文件形式存在，每个文件代表一个完整的功能或内容单元。页面之间无直接链接依赖，可单独部署或引用。

- 每个页面均为自包含的 HTML 文件（含样式与脚本内联或引用公共资源）。
- 页面文件名应尽量反映其内容或功能，便于识别。
- 不强制要求页面间风格统一，但建议保持基本的可读性和兼容性。

## 维护说明

- 本仓库接受新增页面或对现有页面的更新。
- 新增页面请直接提交至根目录，避免创建深层子目录。
- 如页面依赖外部资源（字体库、图标库等），建议使用 CDN 链接或自托管资源。
- 仓库维护者会定期检查页面兼容性，但不保证所有页面在极端老旧浏览器下的可用性。
- 提交前请确保页面无重大显示错误或功能缺失。
